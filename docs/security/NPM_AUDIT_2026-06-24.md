- Complete dependency tree: 34 vulnerabilities: 0 info, 4 low, 12 moderate,
  15 high, 3 critical.
- Production-only tree using `--omit=dev`: 2 vulnerabilities: 0 info, 0 low,
  2 moderate, 0 high, 0 critical.

  and `vitest`. All are development/build tooling dependencies in this

  tree through packages used by the build pipeline.

  build/compiler tooling, not by the generated static runtime.

- No critical production exposure was found for the static GitHub Pages SPA.

  because CI and local development still execute vulnerable packages.
- Deployment may safely remain online while remediation is planned, provided
  only trusted maintainers run local dev/test tooling and CI continues to build
  from reviewed commits.

## Detailed findings

### Production-only findings: `postcss` and `nanoid`

Dependency paths:

- `postcss@8.4.33` is installed directly as a dev dependency and is also used
  by production dependency paths including `vue > @vue/compiler-sfc > postcss`,
  `vite > postcss`, and Tailwind/PostCSS build plugins.
- `nanoid@3.3.7` is reached through `postcss > nanoid`.

Affected functionality:

- CSS parsing/stringification and build/compiler support.
- No imports of `postcss` or `nanoid` were found in `src/`.
- Generated GitHub Pages output is static HTML/CSS/JS. There is no production
  Node startup process.

Exploit conditions:

- `postcss` XSS advisories require unsafe CSS stringification of attacker
  controlled CSS containing crafted style closing sequences.
- `nanoid` advisory requires non-integer size inputs to random ID generation.

Actual project exposure:

- Practical browser runtime exposure is low because the shipped app does not
  execute these packages in the browser.
- Build-time exposure remains real: compromised or malicious CSS/source inputs
  in a reviewed commit could be processed in CI.

Available remediation:

- Update `postcss` to a fixed 8.x release and rerun `npm audit --omit=dev`,
  `npm run build`, `npm run verify:production-build`, and Cypress.
- Verify that generated CSS and Tailwind output remain unchanged except for
  expected tool output ordering/minification differences.

### Critical development finding: `vitest`

Dependency path: `vitest@1.2.1` direct dev dependency.

Affected functionality:

- Vitest API/UI server behavior. The project uses `vitest --run`, not Vitest
  UI, but the package still carries critical advisories in the installed tree.

Exploit conditions:

- A malicious website or local actor would need to interact with a listening
  Vitest API/UI server in vulnerable configurations.

Actual project exposure:

- Not shipped to production. The risk is local development/CI toolchain risk.
- CI runs tests from reviewed code on an ephemeral runner, which limits but
  does not eliminate risk.

Available remediation and risks:

- npm audit suggests `vitest@4.1.9`, a major upgrade.
- The migration may require Vite/Vitest config updates and retesting the Pinia
  store suite under jsdom.

### Critical/high Cypress and request stack

Dependency paths:

- `cypress@13.6.3 > @cypress/request@3.0.1 > form-data@2.3.3`
- `cypress@13.6.3 > @cypress/request@3.0.1 > qs@6.10.4`
- `cypress@13.6.3 > @cypress/request@3.0.1 > uuid@8.3.2`
- `cypress@13.6.3 > lodash@4.17.21`
- `cypress@13.6.3 > tmp@0.2.1`

Affected functionality:

- Cypress internal HTTP/request handling, temporary files, and utility code.

Exploit conditions:

- Primarily local/CI test execution. Exploitation would require malicious test
  code, malicious local endpoints, or crafted data flowing through Cypress
  internals.

Actual project exposure:

- No browser production bundle exposure. Cypress is never deployed.

Available remediation and risks:

- First try a Cypress 13.x minor update allowed by the current range, then
  reassess audit output. If paths remain, plan a Cypress 15 major upgrade.
- Retest all six E2E specs and the custom `scripts/run-cypress-e2e.mjs`
  wrapper.

### High Vite/Rollup/esbuild build-tooling cluster

Dependency paths:

- `vite@5.0.12` direct dev dependency.
- `vite@5.0.12 > rollup@4.9.6`
- `vite@5.0.12 > esbuild@0.19.11`
- `@vitejs/plugin-vue@5.0.3 > vite@5.0.12`
- `vitest@1.2.1 > vite@5.0.12`

Affected functionality:

- Vite dev server file serving, launch-editor integration, bundling, optimized
  dependencies, and Rollup-generated output.

Exploit conditions:

- Mostly dev-server access, crafted local requests, Windows UNC/path handling,
  or malicious build inputs.

Actual project exposure:

- The static deployed app does not include Vite server code.
- Build integrity risk is meaningful because CI generates the deployed assets.

Available remediation and risks:

- Update Vite, `@vitejs/plugin-vue`, and Vitest together in a dedicated
  toolchain migration. Full advisory clearance may require a major Vite line.
- Verify `vite.config.ts`, `vitest.config.ts`, production build output, Pages
  artifact verification, and Cypress.

### High MSW/Tailwind/globbing cluster

Dependency paths include:

- `msw@2.1.3 > chokidar > braces/picomatch`
- `msw@2.1.3 > path-to-regexp`
- `msw@2.1.3 > inquirer > external-editor > tmp`
- `tailwindcss@3.4.1 > micromatch > braces/picomatch`
- `tailwindcss@3.4.1 > postcss-load-config > yaml`

Affected functionality:

- Mock-service-worker tooling, file watching, route pattern matching, and CSS
  content scanning.

Exploit conditions:

- Crafted glob patterns, file paths, or route patterns processed by local tools.

Actual project exposure:

- No MSW runtime is imported by app source.
- Tailwind is build-time only.

Available remediation and risks:

- Update MSW within v2 first and update Tailwind within v3 first; reassess
  whether Tailwind v4 migration is required.
- Tailwind v4 is a major migration and should not be mixed into this runtime
  foundation task.

### High ESLint/test utility cluster

Dependency paths include:

- `eslint@8.57.0 > minimatch`, `ajv`, `js-yaml`
- `eslint@8.57.0 > file-entry-cache > flat-cache > flatted`
- `@vue/test-utils@2.4.3 > js-beautify > editorconfig/minimatch/glob`

Affected functionality:

- Linting, config parsing, cache parsing, and test utility formatting paths.

Actual project exposure:

- Development-only. Not shipped to production.

Available remediation and risks:

- ESLint 8 is behind and current latest is a major version. Plan a separate
  ESLint migration if patch/minor updates do not clear vulnerable paths.
