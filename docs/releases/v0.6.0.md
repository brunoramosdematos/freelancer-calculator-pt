# v0.6.0 — Fiscal confidence and report export

## Production

https://freelancerpt.brunomatos.dev/

## Highlights

- Fiscal-data coverage/status near the tax-year selector.
- About-page fiscal-data coverage and source-reference section.
- Printable income report preview.
- Browser print-to-PDF workflow with no PDF dependency.
- Lazy-loaded chart and secondary routes.
- Reduced initial JavaScript payload.
- Lighthouse performance threshold raised.
- Preferences/hero/action-toolbar UI stability fixes.
- Cross-platform formatting policy.

## What changed

### Product

- Added a fiscal-data status layer near the tax-year selector so users can see
  selected-year coverage, latest supported year information and source context.
- Added fiscal-data coverage and source references to the About page.
- Added a printable income report preview for the active simulation.
- Kept report export in the browser print dialog, allowing users to save as PDF
  without introducing a PDF-generation dependency.

### Performance

- Lazy-loaded the income breakdown chart.
- Lazy-loaded secondary routes so the first simulator load carries less
  JavaScript.
- Reduced the initial JavaScript payload while preserving existing simulator,
  URL and saved-simulation behavior.
- Raised the Lighthouse performance threshold after measuring the optimized
  build.

### Quality

- Prevented Portuguese theme-option labels from overflowing inside the
  preferences panel.
- Stabilized simulator hero spacing between landing and active states.
- Aligned the active simulator action toolbar across supported locales,
  viewports and Dark theme.
- Documented and enforced repository line-ending policy for stable formatting
  checks across Windows and Linux.

### Documentation

- Updated performance documentation for the optimized bundle shape.
- Updated quality/release documentation for the current gates.
- Updated tax-update documentation around fiscal-data coverage and source
  references.
- Preserved security documentation showing production audit and critical audit
  gates remain clean while development-tooling findings are tracked separately.

## Validation

Create this release only after all of the following pass:

- `npm run check:release`;
- the normal GitHub Actions deploy workflow on `main`;
- the tag-triggered release-validation workflow for `v0.6.0`;
- production smoke checks against https://freelancerpt.brunomatos.dev/.

## Known limitations

- Calculations are estimates and do not replace accounting, legal or tax advice.
- The simulator does not model every possible Portuguese tax situation.
- Report export uses the browser print dialog rather than direct binary PDF
  generation.
- The complete npm audit still contains development-tooling moderate/high
  findings, while the production audit and critical audit are clean.
- Lighthouse remains a smoke gate and not a full real-user performance measure.
- Plain `npx tsc --noEmit` is diagnostic-only in this repository; the
  authoritative type gate is `npm run typecheck`.

## Tax disclaimer

This release provides an open-source estimator for freelancer income scenarios
in Portugal. Users should validate their own situation with Autoridade
Tributaria e Aduaneira, Seguranca Social or a qualified professional.
