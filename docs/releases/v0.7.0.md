# v0.7.0 — Scenario comparison

## Production

https://freelancerpt.brunomatos.dev/

## Highlights

- Compare the current simulation against alternative scenarios.
- Compare individual assessment and joint single-income assessment.
- Compare Social Security -20%.
- Compare a no-dependents scenario.
- See gross income, IRS, Social Security, annual net income, monthly net income
  and difference versus the current scenario.
- Use responsive scenario cards instead of a dense comparison table.
- Current simulation, URL, saved simulations and report export remain unchanged
  by comparison actions.

## What changed

### Product

- Added a collapsed comparison panel in the results area.
- Added deterministic scenario presets.
- Added current-baseline card and up to three alternatives.
- Added current, alternative and best-result status chips.
- Added localized better/worse/same labels.
- Added mobile-friendly card layout.

### Quality

- Added Vitest coverage for isolated scenario calculations and mutation safety.
- Added Cypress coverage for presets, no URL mutation, saved-simulation
  invariants, report invariants, locale/theme/mobile behavior and
  accessibility.
- Reworked the comparison layout for readability in light/dark themes and all
  supported locales.

## Validation

Create this release only after all of the following pass:

- `npm run check:release`;
- the normal GitHub Actions deploy workflow on `main`;
- the tag-triggered release-validation workflow for `v0.7.0`;
- production smoke checks against https://freelancerpt.brunomatos.dev/.

## Known limitations

- Comparison is on-screen only in this version.
- Comparison scenarios are not saved to saved simulations.
- Comparison scenarios are not included in share URLs.
- Comparison scenarios are not included in report export yet.
- Custom editable comparison scenarios are not implemented yet.
- Calculations are estimates and do not replace accounting, legal or tax advice.
- The simulator does not model every possible Portuguese tax situation.
- The complete npm audit still contains development-tooling moderate/high
  findings, while the production audit and critical audit are clean.
- Plain `npx tsc --noEmit` is diagnostic-only in this repository; the
  authoritative type gate is `npm run typecheck`.

## Tax disclaimer

This release provides an open-source estimator for freelancer income scenarios
in Portugal. Users should validate their own situation with Autoridade
Tributaria e Aduaneira, Seguranca Social or a qualified professional.
