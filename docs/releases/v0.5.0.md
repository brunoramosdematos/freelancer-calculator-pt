# v0.5.0 - First public release

## Production

- Production URL: https://freelancerpt.brunomatos.dev/
- Release type: first formal public GitHub release.
- Package version: `0.5.0`.

## Highlights

- Estimates Portuguese freelancer IRS, Social Security and net income for tax
  years 2023, 2024, 2025 and 2026.
- Supports individual assessment and joint single-income assessment.
- Supports dependent counts and implemented dependent age groups.
- Provides English, Portuguese (Portugal) and Portuguese (Brazil) interfaces.
- Supports system, light and dark appearance preferences.
- Supports shareable hash-router URLs and browser-local saved simulations.
- Ships with production dependency audit, critical audit, format, lint,
  typecheck, Vitest, Cypress, accessibility, Lighthouse and bundle budget gates.
- Documents release governance, versioning, tax update policy, branch
  protection, quality gates, accessibility, performance, theming and i18n.

## Validation

The release gate is `npm run check:release`. It includes:

- production dependency audit;
- critical dependency audit;
- format check;
- lint;
- Vue-aware typecheck;
- Vitest;
- production build verification;
- bundle budget verification;
- Lighthouse smoke checks;
- Cypress end-to-end tests;
- Cypress axe accessibility checks;
- release-readiness verification.

The release validation workflow runs on `v*.*.*` tags and manual dispatches.

## Known limitations

- Calculations are estimates and do not replace accounting, legal or tax advice.
- The simulator does not model every possible Portuguese tax situation.
- VAT, withholding exemptions and accounting-specific deductions are outside the
  current scope.
- Plain `npx tsc --noEmit` is diagnostic-only in this repository; the
  authoritative type gate is `npm run typecheck`.
- The complete development dependency audit may still report known tooling
  findings. The production dependency audit is the deployment security gate.

## Tax disclaimer

This release provides an open-source estimator for freelancer income scenarios
in Portugal. Users should validate their own situation with Autoridade
Tributaria e Aduaneira, Seguranca Social or a qualified professional.
