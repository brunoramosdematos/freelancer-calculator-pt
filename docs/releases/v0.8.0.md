# v0.8.0 — Joint two-income assessment

## Production

https://freelancerpt.brunomatos.dev/

## Highlights

- Added joint assessment with two incomes.
- Added spouse / partner annual gross income input.
- Uses a simplified second-income model:
  - specific deduction up to EUR 4,104;
  - taxable base = gross income minus the simplified deduction;
  - household taxable income = freelancer taxable base + spouse taxable base;
  - marital quotient applies to household taxable income;
  - Social Security remains based only on freelancer income.
- Added calculation details for the new household model.
- Updated report export to include spouse / partner income when active.
- Updated scenario comparison to support the new scenario.
- Made the default tax year dynamic based on supported tax-data coverage.
- Stabilized local release checks on Windows with a Node-based gate runner.

## Tax model scope

- The second income is modeled as simplified annual gross
  employment/pension-like income.
- The model does not include spouse withholding tax.
- The model does not include spouse Social Security.
- The model does not model separated assessment.
- The model does not model all spouse income categories.
- The result is an estimate, not a full IRS declaration.

## Quality

- Added Vitest and Cypress coverage for calculation, URL hydration, saved
  simulations, report export, scenario comparison, locale/theme/mobile behavior
  and accessibility.
- Hardened the Cypress runner to detect spec-level failures.
- Replaced fragile shell-chained release gates with a Node-based release gate
  runner.

## Validation

Create this release only after all of the following pass:

- the tag-triggered release-validation workflow for `v0.8.0`;

## Known limitations

- Calculations are estimates and do not replace accounting, legal or tax
  advice.
- The second income uses a simplified deduction model.
- The model does not include spouse withholding tax.
- The model does not include spouse Social Security.
- The model does not model separated assessment.
- The model does not model full spouse income categories.
- The complete npm audit still contains development-tooling moderate/high
  findings, while the production audit and critical audit are clean.
