# Internationalization

## Supported Locales

The application supports:

- `en` - English
- `pt-PT` - Portuguese for Portugal

The default and fallback locale is `en`.

## Detection And Persistence

Runtime detection uses this order:

1. Valid stored user preference.
2. `navigator.languages`, in browser preference order.
3. `navigator.language`.
4. `en`.

The localStorage key is:

```text
freelancer-calculator-pt:locale:v1
```

Locale normalization maps `en-*` to `en`, and maps `pt`, `pt-PT`, `pt_PT`,
`pt-BR`, and other `pt-*` values to `pt-PT` for this first phase. `pt-BR`
does not yet have a dedicated catalogue because Brazilian Portuguese and
Brazilian tax terminology will be handled in a later localization phase.

Storage failures are ignored after updating the in-memory Vue I18n locale, so
private browsing or quota errors do not block language switching.

## URLs And Saved Simulations

Locale is intentionally not stored in simulator URLs. Shared calculation URLs
must remain calculation-only and backward-compatible with existing parameters.

Locale is also intentionally not stored in saved simulation objects. Saved
simulations restore tax state only; the interface language remains the user's
current preference.

Resetting the simulator clears calculation state and URL query parameters, but
does not reset the locale preference.

## Catalogue Structure

Messages live under `src/i18n/locales/` and use semantic domain keys. Current
domains include:

- `app`
- `navigation`
- `actions`
- `frequency`
- `units`
- `simulator`
- `assessment`
- `dependents`
- `advancedSettings`
- `results`
- `irsCalculation`
- `socialSecurityCalculation`
- `deductions`
- `chart`
- `simulations`
- `about`
- `footer`
- `metadata`
- `accessibility`
- `validation`

Keys should describe meaning, not English source text. Prefer
`dependents.total.label` over a literal key such as `"Number of dependents"`.
Do not concatenate translated sentence fragments. Use named interpolation and
Vue I18n pluralization for counted nouns.

`src/i18n/messageParity.spec.ts` verifies that English and Portuguese catalogues
have identical keys and no empty messages.

## Metadata

Runtime locale changes update:

- `<html lang>`
- document direction, fixed to `ltr`
- `document.title`
- `meta[name="description"]`

Canonical URL, Open Graph URL, routes, robots.txt, and sitemap.xml remain
unchanged in this phase.

## Portuguese Glossary

Use Portugal-specific terminology:

- Simulator: Simulador
- About: Sobre
- Simulations: Simulações
- Income: Rendimento
- Gross income: Rendimento bruto
- Net income: Rendimento líquido
- Total taxes: Total de impostos
- Income tax year: Ano fiscal
- Tax assessment: Tributação
- Joint - one income: Conjunta - um rendimento
- Social Security: Segurança Social
- Dependent age groups: Faixas etárias dos dependentes
- Aged 3 or under: Até 3 anos
- Aged 4-6: Dos 4 aos 6 anos
- Aged 7+: 7 anos ou mais
- Calculated automatically: Calculado automaticamente
- Paid months per year: Meses remunerados por ano
- Unpaid days off: Dias de ausência não remunerados
- Advanced tax settings: Opções fiscais avançadas
- Professional expenses: Despesas profissionais
- Calculation details: Detalhes do cálculo
- Income breakdown chart: Gráfico da distribuição do rendimento
- Save: Guardar
- Share: Partilhar
- Reset: Repor
- Saved simulation: Simulação guardada
- Indicative estimate: Estimativa indicativa
- About and disclaimer: Sobre e aviso legal

Avoid Brazilian substitutions such as `usuário`, `salvar`, `renda`, `arquivo`,
`INSS`, and `IRPF`.

## Untranslated Legal Terms

Legal acronyms and identifiers remain unchanged where appropriate:

- IRS
- IAS
- RNH
- NHR / RNH
- Categoria B
- recibos verdes
- IVA

Data attributes, URL parameter names, enum values, route names, storage keys,
GitHub URLs, and user-entered simulation names are not translated.

## Formatting

Number, currency, and percentage formatting is intentionally unchanged in this
phase. Locale-specific financial formatting is deferred to a later localization
phase together with future `pt-BR` support.

## Adding A Locale

To add a locale:

1. Add the locale code to `SUPPORTED_LOCALES` in `src/i18n/locale.ts`.
2. Add metadata in `src/i18n/metadata.ts`.
3. Add a complete message catalogue under `src/i18n/locales/`.
4. Register the catalogue in `src/i18n/index.ts`.
5. Add the language to `LocaleSwitcher.vue`.
6. Extend locale detection tests, message parity tests, and Cypress i18n
   coverage.

Do not add localized routes, URL locale parameters, or saved-simulation locale
fields without a separate product decision.
